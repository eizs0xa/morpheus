# example-monorepo-multi-stack — Agent Guide (Claude)

This project uses the **morpheus-platform** (`v0.1.0`).

## Start here

- **Project law:** see [`.agent/constitution.md`](./.agent/constitution.md). Every rule in
  that file is binding for this project. Read it before editing code.
- **Skills catalog:** see [`.agent/skills/`](./.agent/skills/). Invoke skills by name; each
  skill file has frontmatter that declares when to use it and when not to.
- **Feature templates:** see [`.agent/feature-template/`](./.agent/feature-template/) for
  `prd.md`, `spec.md`, `plan.md`, and `tasks.json` stubs.
- **Platform manifest:** see `platform-manifest.json` at the repo root for the installed
  profile, stacks, workspace, PM, and git provider.

## Conventions for agents working on this project

1. Read the constitution first. Do not write code before you have.
2. Cite the constitution, spec, or lore whenever you propose a non-trivial change.
3. Use the skills in `.agent/skills/` — do not reinvent them.
4. Respect composition rules: exactly one workspace, exactly one git provider, 0..1 PM,
   0..N stacks, 0..N domains.
5. Never edit `platform-manifest.json` by hand — run the CLI's update flow.

## When the project changes

If a new stack, workspace, or integration is added, re-run `agentic init --resume` so the
manifest, skills, and templates stay in sync.
